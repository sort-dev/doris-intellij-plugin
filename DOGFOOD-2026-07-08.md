# P3 polish sweep — outcomes (froze-over, 2026-07-08)

1. CAST-targets-colored-like-comments — FIXED. Root cause: the platform SqlSyntaxHighlighter
   builds its highlighting lexer from the ParserDefinition (LanguageParserDefinitions.forLanguage),
   i.e. DorisLexer — the parser-side SQL_BLOCK_COMMENT masks ARE the editor colors (the old
   "separate highlighter lexer" assumption was wrong; every masked span rendered comment-gray, not
   just cast targets). Fix: DorisMaskedSpanRecolorAnnotator lays keyword/identifier/literal
   TextAttributes back over masked ranges (a PsiComment whose text lacks a real comment introducer
   is provably a mask). TextAttributes only — no PSI, parser, or golden change. Pinned in
   DorisMaskedSpanRecolorTest.

2. TVF property-key completion "didn't offer" — FIXED. The provider always fired for both quote
   styles (pinned now for 'single-quoted' too); the real-file failure was AUTO-POPUP: the
   platform's SkipAutopopupInStrings suppresses completion popup inside any string literal, and
   'key' = 'value' operands lex as strings. DorisTvfAutoPopupConfidence (completion.confidence,
   order="first") answers "do not skip" exactly inside a registered TVF call's argument parens.
   Pinned in DorisTableFunctionsTest (completion both quote styles + confidence both ways).

3. LATERAL VIEW POSEXPLODE alias refs red — DEGRADED (suppression), both flag modes. The construct
   has no platform PSI shape; nothing can resolve the aliases (or the generator's argument column,
   which also red-flagged). DorisHighlightInfoFilter rule 5: drop unresolved-reference errors for
   names matching a lateral-view table/column alias declared in the same statement, and for
   references physically inside a lateral-view span (textual gate with a quote-/paren-aware
   scanner, like the EXCEPT gates). Non-alias names outside the construct stay red (pinned both
   ways in DorisHighlightingSuppressionTest). Real resolution = platform lateral-view modeling
   (relic).

4. Inline INDEX name red (item 7) — INVESTIGATED, structural fix DECLINED, still no repro. Light
   fixture (both flag modes, and raw MySQL) shows the shape quiet without a data source — the
   inspection needs an attached model, so the DS-attached repro remains pending. The proposed
   SQL_TABLE_ELEMENT_LIST wrapper is NOT emittable by the replay bridge:
   SqlCompositeElementTypes.SQL_TABLE_ELEMENT_LIST is a SqlLazyElementType chameleon
   (IReparseableElementType) the platform only ever creates COLLAPSED —
   marker.done(type) hard-crashes PSI creation (AssertionError in
   SqlElementFactory.createCompositeElement: MysqlElementFactory has no mapping), and
   marker.collapse(type) would lazy-REPARSE the span with the generated MySQL DDL grammar,
   reintroducing the Doris mis-parses replay avoids ("<type> expected, got 'STRING'") and breaking
   the zero-PsiErrorElement replay contract. Documented in CstReplayer.collect; goldens untouched.
